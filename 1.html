html은 hypertext markup language의 약자로, creating web pages and web applications를 표현하는 주된 언어.

1.1.1 일반 태그

여는 태그와 닫는 태그로 이루어져 있다.

1.1.2 닫지 않는 태그

무엇인가를 설명하지 않는 태그는 감싸야 하는 컨텐츠가 없다. 그래서 닫지 않는다는 규칙.
예) img, input, br, hr, meta 등.

1.1.3 Paragraph

문단을 표현하는 태그는 하나의 단락을 그룹핑 할 수 있도록 열고 닫는 태그가 존재한다.
br이 줄바꿈만을 의미하는 태그라면, p태그는 하나의 단락으로 묶어주는 역할을 한다.
다만, 단락 간 간격이 고정되어 있으므로 시각적 자유도는 떨어지는 편.
But! CSS라는 기술이 있다. html과 다른 문법을 가진 언어인데, 정보를 꾸며준다. (html은 정보를 표현하는 언어)


다시말하면,

p 태그에 style="margin-top:45px" 추가하면 여백 생김.(CSS)
p태그를 통해 단락의 경계를 분명히 하면서 CSS를 사용한 p태그 디자인을 자유롭게 변경할 수 있기 때문에 br태그보다는 p태그가 더 좋은 선택.

1.2 HTML의 중요성

웹의 핵심 철학은 접근성(accessibility).
웹은 모든 운영체계에서 동작하고, 웹의 소스코드는 누구나 볼 수 있고, 웹은 저작권 없는 순수 공공재.(다른 기술들과의 차별성)
HTML을 의미론적으로 잘 사용한다면 소외되는 사람 없이 누구나 접근가능함.



2. 속성(attribute)

html의 기본문법을 완전히 마스터 한다!
아주 인기있는 태그 : img 태그
이미지 태그를 사용한다고 하여 바로 나오는 것이 아니다. 왜?
어떤 이미지를 불러올 지에 대한 정보 부족.
새로운 문법 도입 : 속성


여기서 속성은 src. https://이하는 이미지의 주소.
따라서 src라는 속성을 통해 이미지를 불러올 수 있게 된 것.

내 컴퓨터에 있는 이미지를 표시하려면?
1. 이미지 파일을 웹페이지 파일과 같은 위치에 놓는다.
2. 이미지 파일의 이름을 src 속성의 값으로 적어준다.
예) img src = "la_mer.jpg"

앗. 사이즈가 너무 크다.
width로 조정하자.(50%)
